문법 드릴만 반복하니까 실감이 안 났어요. 어차피 뒤 커리큘럼에서 로봇 시연 데이터를 다루게 될 텐데, 그 형식을 미리 만져보면 문법 연습도 실전이 되겠다 싶었어요. 그래서 공개 로봇 데이터셋을 재료로 연습 문제를 만들었어요.

## 어떤 데이터인가요

HuggingFace에 공개된 [`lerobot/pusht`](https://huggingface.co/datasets/lerobot/pusht)예요. 로봇이 T자 블록을 미는 시연(demonstration)을 모아둔 모방학습(Imitation Learning)용 데이터셋이에요.

- 시연 206개(에피소드), 프레임 25,650개, 10fps
- 수치 데이터는 Parquet, 카메라 영상은 MP4, 메타데이터는 JSON
- 각 프레임은 관측(observation)과 행동(action)의 쌍 — 모방학습은 이 쌍으로 "이 상황에서 이렇게 움직여라"를 배워요

## 문서보다 데이터가 맞아요

드릴을 만들면서 처음 걸린 건 구조였어요. 자료에서 봤던 구조는 에피소드마다 parquet 파일이 하나씩 있는 형태였는데, 실제 데이터셋은 포맷이 v3.0으로 개편되면서 청크 파일 하나에 에피소드 206개가 통합돼 있었어요. 오픈소스 로보틱스 스택은 개정이 빨라서, 몇 달 전 자료도 이미 과거형이 되더라고요.

여기서 원칙 하나를 얻었어요. **문서와 데이터가 다르면, 데이터가 맞다.** 구조는 문서로 배우되, 확정은 실물을 열어서 해야 해요.

## 0.1이 0.1이 아니었어요

검증 문제를 만들다 하나 더 걸렸어요. 10fps 데이터니까 타임스탬프 간격은 0.1초여야 하고, `간격 == 0.1`로 검사하면 될 줄 알았는데 실패했어요.

원인은 저장 타입이었어요. timestamp가 **float32**로 저장돼 있어서 0.1이 `0.10000000149…`로 들어와 있었고, 배정밀도(float64) 상수 `0.1`과의 `==` 비교가 깨진 거예요. 그래서 검사를 이렇게 바꿨어요.

```python
math.isclose(gap, 0.1, abs_tol=1e-6)  # 절대 오차 허용으로 비교
```

교과서 예제에서는 안 만나는, 진짜 센서 데이터라서 만나는 종류의 함정이에요. 실물 데이터는 항상 어딘가가 이상적이지 않다는 걸 몸으로 확인했어요.

## 드릴은 이렇게 생겼어요

8문제예요. 전부 "데이터를 믿을 수 있는지 검증하는" 관점의 문제로 만들었어요.

- 관측 키만 골라내기, 에피소드별 프레임 수 집계, 최장 에피소드 찾기
- 인접 타임스탬프 간격 검사(위 float32 함정이 여기서 나와요)
- 행동 좌표의 범위 확인, 에피소드 경계 무결성 — 에피소드가 바뀔 때 frame_index가 0으로 리셋되는지

재현 장치도 넣었어요. 데이터셋의 **리비전(커밋 해시)을 고정**해서 나중에 상류 데이터가 개정돼도 드릴은 같은 데이터로 재현되고, 문제마다 assert 셀프체크가 붙어 있어서 실행하면 통과/실패가 바로 찍혀요. 답안 파일은 8문제 전부 실행 검증을 통과해 둔 상태예요.

```
로드 완료: lerobot/pusht v3.0 - 206 episodes / 25650 frames
...
결과: 8/8
```

## 마치며

파이썬 문화를 배우고([[2026-07-07_회고_파이썬_다시배우기]]), 그 문법으로 진짜 로봇 데이터를 뜯는 문제를 만들었어요. 문법 → 실데이터 → 검증 관점까지 한 줄로 이어진 게 이번 주의 수확이에요. 데이터를 더 깊게 다루는 시점이 오면, 이 드릴부터 다시 풀면서 시작하려고 해요.
