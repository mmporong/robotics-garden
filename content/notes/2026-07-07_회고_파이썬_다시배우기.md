게임 개발을 하다가 로보틱스로 방향을 틀었어요. 부트캠프 첫 모듈은 C++와 파이썬인데요. 솔직히 파이썬 파트는 만만하게 봤어요. 5년 넘게 C#을 썼고, 파이썬도 읽을 줄은 아니까요.

그런데 하루를 통으로 쓰고 나서야 알았어요. 제가 몰랐던 건 문법이 아니라 **문화**였다는 걸요.

## 전부 외우지 않기로 했어요

처음엔 문법책 순서대로 정리하다가 중간에 멈췄어요. 이렇게 정리하면 한 달 뒤에 아무것도 안 남는다는 걸 경험으로 알거든요. 그래서 그날 배운 걸 전부 꺼내놓고 **현업에서 쓰는 빈도순**으로 다시 줄을 세웠어요.

- **Tier 1 — 매일 쓰는 것**: f-string, 컴프리헨션, `dict.get`, `enumerate`/`zip`, `with`, `join`, 언패킹, truthiness, `sorted(key=...)`. 딱 아홉 개예요. 이건 안 보고 손에서 나와야 해요.
- **Tier 2 — 주 단위로 쓰는 것**: `deque`, `Counter`, `heapq`, `pathlib`, 구체 예외 처리, 클래스 상속, 타입 힌트. 형태만 기억하면 돼요.
- **Tier 3 — 함정 회피**: 쓰는 게 아니라 **피하는** 지식이에요. 아는 것 자체가 실력인 목록이죠.

우선순위를 매기고 나니 마음이 편해졌어요. Tier 1 아홉 개만 손에 익으면, 나머지는 필요할 때 찾으면 되니까요.

## C# 개발자의 직관이 배신당한 지점들

### 변수는 상자가 아니라 이름표였어요

혹시 이런 코드를 보신 적 있나요?

```python
b = a
b.append(1)
print(a)  # a도 바뀌어 있어요
```

C#은 값 타입과 참조 타입을 구분해서 생각하는 언어예요. 파이썬은 그 구분이 없어요. **모든 변수가 객체에 붙이는 이름표**고, `b = a`는 복사가 아니라 같은 객체에 이름을 하나 더 붙이는 행위예요. 머리로는 알고 있었는데요. "복사가 필요하면 `a.copy()`를 명시한다"가 아직 손에 배어 있지 않다는 걸 인정해야 했어요.

### 가변 기본값 인자 — C#에는 없는 종류의 함정이에요

```python
def append_log(item, logs=[]):   # 함정이에요
    logs.append(item)
    return logs
```

기본값이 **함수를 정의하는 시점에 딱 한 번** 평가돼서, 모든 호출이 같은 리스트를 공유해요. 증상은 괴담처럼 나타나요. "함수를 두 번 불렀는데 첫 호출의 데이터가 남아 있어요."

C#의 기본값 매개변수는 컴파일 타임 상수만 허용해서 이 함정 자체가 존재하지 않아요. 파이썬에서는 `logs=None`으로 받고 함수 안에서 새로 만드는 게 관용구예요.

### 컴프리헨션의 if는 위치가 두 개예요

```python
[x for x in nums if x > 0]          # if가 뒤 = 필터
[x if x > 0 else 0 for x in nums]   # if-else가 앞 = 값 변환
```

LINQ로 치면 앞은 `Where`, 뒤는 `Select`에 삼항을 넣은 거예요. 같은 `if`인데 위치에 따라 역할이 달라진다는 게 처음엔 어색했는데요. "필터는 뒤, 변환은 앞"으로 외우니까 정리됐어요.

### 예외를 대하는 태도가 반대예요

C#에서는 예외가 비싸니까 미리 검사하라고 배웠어요. 파이썬 문화는 반대였어요. **"허락을 구하지 말고 용서를 구하라"(EAFP)**예요.

`if key in d:`로 검사하고 꺼내는 대신, 일단 꺼내고 `except KeyError:`로 받거나 아예 `d.get(key, default)`로 끝내요. 검사와 사용 사이에 상태가 바뀌는 경쟁 조건까지 원천 차단된다는 설명을 듣고 납득했어요. 문법이 아니라 철학이 다른 거였어요.

### 반가운 것도 있었어요

`with open(path) as f:`는 C#의 `using`과 정확히 같은 물건이에요. 블록을 벗어나면, 예외가 나도, 자원이 해제돼요.

로봇 쪽에서는 이게 파일보다 **시리얼 포트**에서 중요하다고 해요. 포트를 안 닫고 죽은 프로세스 때문에 "포트가 이미 사용 중" 에러로 재접속이 막히는 게 전형적인 사고 패턴이라고 하네요.

## 오늘 배운 문법이 그대로 로봇 코드가 돼요

가장 동기부여가 된 발견은 이거예요. 파이썬 클래스 상속을 배우면서, **프레임워크가 상속을 요구하는 대표적인 예가 ROS2**라는 걸 알게 됐어요.

```python
class MyNode(Node):        # ROS2 노드
class Net(nn.Module):      # PyTorch 모델
```

다음 모듈에서 쓸 ROS2 노드도, 나중에 쓸 PyTorch 모델도 전부 오늘 배운 "상속받고 `__init__` 오버라이드하고 메서드 다는" 그 패턴이에요. 기초 문법이 커리큘럼 전체의 뼈대와 이어져 있다는 걸 확인하면, 지금 하는 반복이 덜 지루해져요.
