문법 드릴만 반복하니까 실감이 안 났어요. 어차피 뒤 커리큘럼에서 로봇 시연 데이터를 다루게 될 텐데, 그 형식을 미리 만져보면 문법 연습도 실전이 되겠다 싶었어요. 그래서 공개 로봇 데이터셋을 재료로 연습 문제를 만들어서 풀었어요.

## 📦 어떤 데이터인가요

HuggingFace에 공개된 [`lerobot/pusht`](https://huggingface.co/datasets/lerobot/pusht)예요. 로봇이 T자 블록을 미는 시연(demonstration)을 모아둔 모방학습(Imitation Learning)용 데이터셋이에요.

- 시연 206개(에피소드), 프레임 25,650개, 10fps
- 수치 데이터는 Parquet, 카메라 영상은 MP4, 메타데이터는 JSON
- 각 프레임은 관측(observation)과 행동(action)의 쌍 — 모방학습은 이 쌍으로 "이 상황에서 이렇게 움직여라"를 배워요

## 🔍 문서보다 데이터가 맞아요

드릴을 만들면서 처음 걸린 건 구조였어요. 자료에서 봤던 구조는 에피소드마다 parquet 파일이 하나씩 있는 형태였는데, 실제 데이터셋은 포맷이 v3.0으로 개편되면서 청크 파일 하나에 에피소드 206개가 통합돼 있었어요. 오픈소스 로보틱스 스택은 개정이 빨라서, 몇 달 전 자료도 이미 과거형이 되더라고요.

여기서 원칙 하나를 얻었어요. **문서와 데이터가 다르면, 데이터가 맞다.** 구조는 문서로 배우되, 확정은 실물을 열어서 해야 해요.

## 🎲 0.1이 0.1이 아니었어요

검증 문제를 만들다 하나 더 걸렸어요. 10fps 데이터니까 타임스탬프 간격은 0.1초여야 하고, `간격 == 0.1`로 검사하면 될 줄 알았는데 실패했어요.

원인은 저장 타입이었어요. timestamp가 **float32**로 저장돼 있어서 0.1이 `0.10000000149…`로 들어와 있었고, 배정밀도(float64) 상수 `0.1`과의 `==` 비교가 깨진 거예요. 그래서 검사를 이렇게 바꿨어요.

```python
math.isclose(gap, 0.1, abs_tol=1e-6)  # 절대 오차 허용으로 비교
```

교과서 예제에서는 안 만나는, 진짜 센서 데이터라서 만나는 종류의 함정이에요.

## 🛠️ 드릴은 이렇게 생겼어요

처음엔 데이터 검증 관점의 8문제로 시작했다가, 문법 연습(컴프리헨션·Counter·deque 같은 것들)과 합쳐 최종 12문제가 됐어요.

- 관측 키만 골라내기, 에피소드별 프레임 수 집계·그룹핑, 프레임 수 랭킹
- 인접 타임스탬프 간격 검사(위 float32 함정이 여기서 나와요)
- 행동 좌표의 범위 확인, 센서 버퍼 이동평균, 통신 두절(None)과 수신 대기(빈 값) 구분

재현 장치도 넣었어요. 데이터셋의 **리비전(커밋 해시)을 고정**해서 나중에 상류 데이터가 개정돼도 드릴은 같은 데이터로 재현되고, 문제마다 assert 셀프체크가 붙어 있어서 실행하면 통과/실패가 바로 찍혀요. 오늘 12문제를 풀어 전부 통과했어요.

```
로드 완료: lerobot/pusht v3.0 - 206 episodes / 25650 frames
  [7 isclose 검증] 통과
  ...
결과: 12/12
```

드릴 전체(문제 58개 + 답안)는 [robotics-python-drills](https://github.com/mmporong/robotics-python-drills)에 공개해 뒀어요. 리비전 고정 덕에 누가 언제 받아도 같은 데이터로 재현됩니다.

## 💥 그런데, 임포트했더니 코드가 실행됐어요

드릴과 별개로 오늘 사건이 하나 있었어요. 클래스를 모듈로 쓰려고 파일을 만들고 아래쪽에 테스트용 print를 뒀는데, 다른 스크립트에서 임포트하는 순간 **그 print가 그냥 실행됐어요.**

원인은 임포트에 대한 제 이해가 틀려 있었던 거예요. 파이썬의 import는 선언만 읽어가는 게 아니라 **대상 파일의 최상위 코드를 위에서 아래로 전부 실행**해요. 그래서 파이썬 파일들이 약속처럼 이 관용구를 쓰는 거였어요.

```python
if __name__ == '__main__':
    # 직접 실행할 때만 도는 코드
```

원리는 변수 하나예요. 파일을 **직접 실행**하면 `__name__`이 `'__main__'`이고, **임포트되면 모듈(파일) 이름**이 돼요. 재미있는 건, 오늘 푼 드릴이 이미 이 구조를 쓰고 있었다는 것 — 드릴 하나가 다른 드릴에서 데이터 로더를 임포트하는데 그때 상대 파일의 채점 코드가 안 돌아가는 이유가 정확히 이 가드였어요. 쓰면서도 몰랐던 걸 문제를 겪고 나서야 알았네요.

## 🤫 오늘 넘어진 것들의 공통점 — 파이썬은 조용해요

드릴을 풀면서 몇 번 넘어졌는데, 돌아보니 전부 같은 성격이었어요.

- `return print(...)`를 썼다가 — print는 화면에 찍고 **조용히 None을 반환**해요
- `min(xs, ys)`를 썼다가 — 인자를 두 개 주면 각각의 최솟값이 아니라 **두 튜플끼리의 비교**가 돼요
- 컴프리헨션에 for문의 append를 섞었다가 — 세 칸 구조(`[내보낼값 for 변수 in 반복대상 if 조건]`)로 다시 조립했어요
- 그리고 임포트 사건과, 위의 float32도 같은 얼굴이에요 — 값이 조용히 달라져 있죠

**파이썬은 조용해요.** 임포트가 조용히 코드를 실행하고, return 없는 함수가 조용히 None을 돌려주고, 대입이 조용히 원본을 공유해요. 정적 타입 언어라면 컴파일러가 소리쳤을 지점들을 에러 없이 지나가고 결과로만 알려줘요. 그래서 `__main__` 가드 같은 관용구와 assert 셀프체크 같은 검증 장치가 이 언어의 문화라는 걸 몸으로 이해했어요.

## 📌 남은 숙제

솔직하게 적어두면 — 컴프리헨션 조립과 `defaultdict`는 아직 백지에서 바로 안 나와요. 주말에 자료 안 보고 다시 푸는 재현 측정에서 이 둘을 1순위로 돌릴 예정이에요.

## 마치며

문법을 배우고, 그 문법으로 진짜 로봇 데이터를 뜯고, 뜯다가 언어의 성격까지 만난 하루였어요. 수업 뒤편에서는 "셀프어텐션"이라는 단어도 지나갔는데 — 트랜스포머의 심장이고 로봇 학습 모델들이 그 위에 서 있다는 것까지만 잡아뒀어요. 제대로 만나는 날 다시 쓸게요.
