로보틱스를 공부하기 시작하면서 파이썬을 다시 잡았어요. 읽을 줄 아는 언어라 만만하게 봤는데, 하루를 통으로 쓰고 나서야 알았어요. 제가 몰랐던 건 문법이 아니라 **문화**였다는 걸요.

## 전부 외우지 않기로 했어요

문법책 순서대로 정리하다가 중간에 멈췄어요. 이렇게 정리하면 한 달 뒤에 아무것도 안 남는다는 걸 경험으로 알거든요. 그래서 그날 배운 걸 전부 꺼내놓고 **현업에서 쓰는 빈도순**으로 다시 줄을 세웠어요.

- **Tier 1 — 매일 쓰는 것**: f-string, 컴프리헨션, `dict.get`, `enumerate`/`zip`, `with`, `join`, 언패킹, truthiness, `sorted(key=...)`. 딱 아홉 개예요. 안 보고 손에서 나와야 하는 목록이에요.
- **Tier 2 — 주 단위로 쓰는 것**: `deque`, `Counter`, `pathlib` 같은 표준 라이브러리. 형태만 기억하면 돼요.
- **Tier 3 — 함정 회피**: 쓰는 게 아니라 **피하는** 지식. 아는 것 자체가 실력인 목록이에요.

우선순위를 매기고 나니 마음이 편해졌어요. Tier 1만 손에 익으면 나머지는 필요할 때 찾으면 되니까요.

## 익숙한 직관이 배신당한 지점들

### 변수는 상자가 아니라 이름표였어요

```python
b = a
b.append(1)
print(a)  # a도 바뀌어 있어요
```

값 타입과 참조 타입을 구분하는 언어(C# 등)에 익숙하면 여기서 한 번 걸려요. 파이썬은 그 구분이 없어요. **모든 변수가 객체에 붙이는 이름표**고, `b = a`는 복사가 아니라 같은 객체에 이름을 하나 더 붙이는 행위예요. 복사가 필요하면 `a.copy()`를 명시해야 해요.

### 가변 기본값 인자라는 함정

```python
def append_log(item, logs=[]):   # 함정이에요
    logs.append(item)
    return logs
```

기본값이 **함수를 정의하는 시점에 딱 한 번** 평가돼서, 모든 호출이 같은 리스트를 공유해요. 증상은 괴담처럼 나타나요. "함수를 두 번 불렀는데 첫 호출의 데이터가 남아 있어요." 해법은 `logs=None`으로 받고 함수 안에서 새로 만드는 거예요.

### 컴프리헨션의 if는 위치가 두 개예요

```python
[x for x in nums if x > 0]          # if가 뒤 = 필터
[x if x > 0 else 0 for x in nums]   # if-else가 앞 = 값 변환
```

같은 `if`인데 위치에 따라 역할이 달라져요. "필터는 뒤, 변환은 앞"으로 외우니까 정리됐어요.

### 예외를 대하는 태도가 반대예요

예외는 비싸니까 미리 검사하라고 배워온 쪽이라면, 파이썬 문화는 반대예요. **"허락을 구하지 말고 용서를 구하라"(EAFP)**. `if key in d:`로 검사하고 꺼내는 대신 일단 꺼내고 `except KeyError:`로 받거나, 아예 `d.get(key, default)`로 끝내요. 검사와 사용 사이에 상태가 바뀌는 경쟁 조건까지 원천 차단된다는 설명을 듣고 납득했어요.

### 반가운 것도 있었어요

`with open(path) as f:`는 C#의 `using`과 같은 물건이에요. 블록을 벗어나면, 예외가 나도, 자원이 해제돼요. 로봇 쪽에서는 이게 파일보다 **시리얼 포트**에서 중요하다고 해요. 포트를 안 닫고 죽은 프로세스 때문에 재접속이 막히는 게 전형적인 사고 패턴이라고 하네요.

## 오늘 배운 문법이 그대로 로봇 코드가 돼요

가장 동기부여가 된 발견이에요. 파이썬 클래스 상속을 배우면서, **프레임워크가 상속을 요구하는 대표적인 예가 ROS2**라는 걸 알게 됐어요.

```python
class MyNode(Node):        # ROS2 노드
class Net(nn.Module):      # PyTorch 모델
```

앞으로 쓸 ROS2 노드도 PyTorch 모델도, 오늘 배운 "상속받고 `__init__` 오버라이드하고 메서드 다는" 그 패턴이에요. 기초 문법이 커리큘럼 전체의 뼈대와 이어져 있다는 걸 확인하면 지금 하는 반복이 덜 지루해져요.

## 정리만 하면 안 남아서, 드릴을 만들었어요

정리 노트는 쓰는 순간에만 뿌듯하더라고요. 그래서 노트와 별도로 **assert 셀프체크가 달린 드릴 파일**을 만들었어요. 문제는 교과서 예제 대신 로봇 시나리오로요. 예를 들면 이런 문제예요.

```python
def next_command(queue):
    """queue가 None이면 'ERROR', 빈 리스트면 'IDLE',
    아니면 첫 명령을 꺼내(pop(0)) 반환. None과 빈 값을 구분할 것."""
```

통신이 끊긴 상태(None)와 할 일이 없는 상태(빈 리스트)를 구분하는 truthiness 문제인데, 로봇 콜백에서 헷갈리면 그대로 버그가 되는 지점이에요. 이런 문제들의 TODO를 채워 실행하면 assert가 문제마다 통과/실패/미시도를 찍어줘요. 기준은 자료 안 보고 전부 통과 — 주말에 처음부터 다시 푸는 게 진짜 측정이에요.

## 마치며

문법은 하루면 훑지만 문화가 손에 배는 건 다른 문제라, 위의 드릴로 반복하고 있어요. 다음은 C++이에요. GC가 치워주는 세계에서 "대입은 복사고, 수명은 내 책임"인 세계로 가요.

그리고 여기서 배운 문법으로 진짜 로봇 데이터를 뜯어보는 문제도 만들었는데, 그 이야기는 [[2026-07-09_로봇데이터_드릴]]에 이어서 썼어요.
