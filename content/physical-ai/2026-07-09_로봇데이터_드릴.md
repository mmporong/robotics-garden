문법 드릴만으로는 실제 데이터의 형태를 알기 어려워서, 뒤 커리큘럼에서 다룰 로봇 시연 데이터를 미리 열어 검증 드릴 12문제를 만들었어요. 그 과정에서 교과서 예제로는 안 드러나는 두 지점, 부동소수점(float32) 비교와 파이썬 import의 실행 모델을 정리하게 됐어요.

## 재료: lerobot/pusht 데이터셋

HuggingFace에 공개된 [`lerobot/pusht`](https://huggingface.co/datasets/lerobot/pusht)를 썼어요. 로봇이 T자 블록을 미는 시연(demonstration)을 모은 모방학습(imitation learning)용 데이터셋이에요.

- 에피소드 206개, 프레임 25,650개, 10fps
- 수치는 Parquet, 영상은 MP4, 메타데이터는 JSON
- 각 프레임은 관측(observation)과 행동(action)의 쌍이고, 모방학습은 이 쌍에서 "이 관측이면 이 행동"을 지도학습으로 배워요.

## 데이터 구조는 실물로 확인해요

참고 자료는 에피소드마다 Parquet 파일이 하나씩 있는 구조라고 했는데, 실제 데이터셋은 v3.0으로 개편되면서 청크 파일 하나에 206개가 통합돼 있었어요. 오픈소스 로보틱스 스택은 개정이 잦아 몇 달 전 자료도 어긋나기 쉬우니, 구조는 문서로 익히되 최종 확인은 실물을 열어서 해요. 문서와 데이터가 다르면 데이터가 맞으니까요.

## float32 타임스탬프의 == 비교 오류

10fps 데이터니까 인접 프레임의 타임스탬프 간격은 0.1초여야 하는데, `간격 == 0.1`로 검증했더니 전부 실패했어요.

원인은 부동소수점 표현이에요. 0.1은 2진수로 유한하게 떨어지지 않아(0.0001100110011… 무한 반복) 어떤 타입도 정확히 담지 못하고 근사하는데, 근사에 쓰는 비트 수가 타입마다 달라요. timestamp는 float32(가수 23비트)라 0.1이 `0.10000000149…`가 됐고, 파이썬 리터럴 `0.1`은 float64(가수 52비트)라 더 촘촘한 다른 근사값이에요. 두 값이 다르니 `==`가 깨져요.

그래서 부동소수점은 정확히 같은지가 아니라 허용 오차 안에서 같은지로 비교해요.

```python
import math
gap == 0.1                            # 실패: 정확 비교
math.isclose(gap, 0.1, abs_tol=1e-6)  # 수정: 허용 오차
```

`abs_tol`은 절대 허용 오차예요. `isclose`는 기본적으로 상대 오차(`rel_tol`)로 비교하는데, 0에 가까운 값에서는 그것만으로 부족해서 절대 오차를 함께 줘요.

## 드릴 구성과 재현 장치

데이터 검증(관측 키 추출, 프레임 수 집계, 타임스탬프 간격 검증, 좌표 범위 확인)과 문법 연습(컴프리헨션·`Counter`·`deque`)을 합쳐 12문제를 만들었어요. 재현을 위해 데이터셋 리비전(커밋 해시)을 고정했고, 문제마다 붙인 assert가 실행 즉시 통과/실패를 찍어요. 전체 드릴(58문제 + 답안)은 [robotics-python-drills](https://github.com/mmporong/robotics-python-drills)에 공개해 뒀어요.

## import는 모듈의 최상위 코드를 실행해요

모듈을 하나 만들고 아래쪽에 테스트용 `print`를 뒀는데, 다른 파일에서 import하는 순간 그 `print`가 실행됐어요.

파이썬의 import는 대상 모듈의 최상위 코드를 위에서 아래로 한 번 실행해요. 함수·클래스 정의문은 물론, 그 사이에 놓인 일반 문장(예: `print`)도 똑같이 실행되죠. 그래서 최상위에 부작용(side effect) 코드를 두면 import만 해도 돌아가요. 한 번 실행된 모듈은 `sys.modules`에 캐시돼서 다시 import해도 재실행되지는 않아요.

직접 실행할 때만 도는 코드는 이 가드로 분리해요.

```python
if __name__ == '__main__':
    # 파일을 직접 실행할 때만 도는 코드
```

`__name__`은 파일을 `python x.py`로 직접 실행하면 `'__main__'`, 다른 파일이 `import x`로 부르면 모듈 이름 `'x'`가 돼요. 그래서 가드 안의 코드는 직접 실행에서만 돌고 import에서는 건너뛰어요.

## 조용한 실패: 예외 없이 값만 달라지는 실수들

드릴에서 반복해 걸린 실수들의 원인이 하나로 모여요. 파이썬은 오류로 멈추지 않고 조용히 다른 값을 돌려준다는 점이에요.

- `return print(...)`: `print`는 값을 반환하지 않아(`None` 반환) 받으면 `None`이 와요.
- `min(xs, ys)`: 인자를 둘 주면 각 리스트의 최솟값이 아니라 두 시퀀스를 사전순 비교한 결과예요. 리스트 안 최소는 `min(xs)`로 넘겨야 해요.
- 컴프리헨션에 `append` 혼용: `[식 for 변수 in 반복 if 조건]` 세 자리를 지키면 `append`가 필요 없어요.

float32 비교와 import 부작용도 같은 얼굴이에요. 정적 타입 언어라면 컴파일 단계에서 걸렸을 것들이 파이썬에서는 런타임에 조용히 통과하니, `__main__` 가드나 assert 셀프체크 같은 방어 관용구가 자주 쓰여요.
